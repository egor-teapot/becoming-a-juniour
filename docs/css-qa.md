## В html есть элементы 3 уровней, какие у них свойства?
block
- всегда занимает 100% ширины родительского контейнера
- выстраивается один под другим
- изменение анонимного контейнера влияют на
позиционирование других элементов

Примеры block элементов: p h1..6 div
table ul ol form

<br>

inline
- занимают только обьем своего наполнения
- выстраивааются в линию, друг после перед другом
- изменение анонимного контейнера не влияют на
позиционирование других элементов

Примеры inline элементов: a butto> code

<br>

inline-block
- занимают только обьем своего наполнения
- выстраивааются в линию, друг после перед другом
- изменение анонимного контейнера влияют на
позиционирование других элементов 

Примеры inline-block элементов: img video

## Для чего используется свойтсво flex?
Как правило используется для сокращения
принемает следующие значения:
- flex-grow
- flex-shrink
- flex-basis

<br>

## Для чего используется flex-grow
Определяет сколько оставшегося места в контейнере
будет отдано под элемент

- 0 - элемент будет занимать наименьшее пространство в контейнере
- 1 - обычное значение flex-grow для элемента
- \>1 - увеличивает колличество занимаемого места в зависимости
от изначального размера элемента

<br>

## Для чего используется flex-shrink?
Определяет насколько элемент будет сжиматся
в зависимости от оставшегося размера в контейнере.
Если контейнер меньше размера элемента, элемент
будет сжиматся в зависимости от указанного flex-shrink

- 0 - элемент не сжимается в зависимости от оставшегося
места в контейнере.
- 1 - обычное значение flex-shrink
- \>1 - элемент сжимается сильнее в зависимости от
изначального размера элемента

<br>

## Для чего используется flex-basis?
Определяет основной размер элементов в зависимости
от оставшегося места в контейнере.

- 0 - элемент занимает размер своего наполнения
- auto - обычное значение для flex-basis
- 100% - установит максимальный размер
равный flex-grow

## Как padding и скролл по ссылкам может сломать верстку?
padding довольно удобное свойство, оно позволяет
создавать пространство между контейнерами не влияя на верстку,
это может быть полезно при создании модулей которые можно будет
добавлять на страницы, однако иногда padding может вести себя
странно.

Так например в моем случае было так, я сделал flex контейнер
и сделал в нем еще два, первый бый сайдбаром второй основным.
В сайдбаре находились ссылки которыми я перемещался по тексту
на сайте. Отступ с верху был сделан при помощи padding.
Из за этого при переходе по некоторым ссылкам отступ с верху
мог становится меньше, иногда немного, а иногда съедало пол
оглавления. Эта проблема была решена созданием обертки к
которой применялся margin-top для создания отступа.

Мне сейчас тяжеле сказать чем конкретно была вызвана эта проблема,
движком браузера или особеностями расчета отступа в самом padding,
но проблема решена и я этому очень рад(+1 в копилку знаний о багах css).
